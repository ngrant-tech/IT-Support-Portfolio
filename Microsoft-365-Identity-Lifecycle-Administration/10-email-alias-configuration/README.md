# Email Alias Configuration

## Overview
Demonstrated hands-on experience creating and assigning an email alias to an existing user account through the Microsoft 365 Admin Center. This process involved accessing user identity settings, configuring an additional email alias, saving administrative changes, and validating alias functionality through successful email delivery testing.

This project demonstrates practical experience with Microsoft 365 email alias management, user identity configuration, and administrative validation in a cloud-based identity management environment.

---

## Environment / Tech Stack
- Microsoft 365 Admin Center
- Microsoft Entra ID
- Exchange Online
- User Account Administration
- Email Alias Management

---

## Email Alias Administration
- Accessed **Active Users** within Microsoft 365 Admin Center
- Selected an existing user account
- Opened **Manage Username and Email**
- Created an additional email alias under the user account
- Saved administrative changes
- Validated alias functionality through email delivery testing

---

## Key Skills Demonstrated
- Microsoft 365 Administration
- Email Alias Management
- User Identity Administration
- Exchange Online Administration
- Administrative Change Validation
- Email Routing Verification

---

## Key Takeaways
Email alias administration is an important function in enterprise identity and communication management. Proper alias configuration allows organizations to support alternate communication channels, naming standardization, and flexible email routing while maintaining centralized account management.
