# Self-Service Password Reset Configuration

## Overview
Demonstrated hands-on experience enabling **Self-Service Password Reset (SSPR)** through Microsoft 365 and Microsoft Entra ID. This process involved configuring organizational password reset policies, enabling password reset functionality for all users, and validating the configuration by simulating a user-driven password recovery workflow using multi-factor authentication (MFA).

This project demonstrates practical experience with Microsoft 365 security administration, password recovery configuration, identity verification, and user account recovery workflows in a cloud-based identity management environment.

---

## Environment / Tech Stack
- Microsoft 365 Admin Center
- Microsoft Entra ID
- Azure Portal
- Microsoft Authenticator
- Identity and Access Management (IAM)
- Self-Service Password Reset (SSPR)

---

## Self-Service Password Reset Administration
- Accessed **Org Settings** within Microsoft 365 Admin Center
- Navigated to **Security & Privacy**
- Opened **Self-Service Password Reset** settings
- Accessed Microsoft Entra ID password reset properties through Azure Portal
- Modified **Self-Service Password Reset Enabled** for all users
- Validated password reset functionality through end-user recovery testing

---

## Key Skills Demonstrated
- Microsoft 365 Administration
- Microsoft Entra ID Administration
- Self-Service Password Reset (SSPR)
- Identity and Access Management (IAM)
- Multi-Factor Authentication (MFA)
- Password Recovery Administration
- User Authentication Validation
- Security Policy Configuration

---

## Key Takeaways
Self-Service Password Reset improves organizational efficiency by reducing help desk dependency for password recovery while maintaining identity verification through MFA. Proper SSPR configuration strengthens account recovery processes and improves user account resilience within cloud identity environments.
