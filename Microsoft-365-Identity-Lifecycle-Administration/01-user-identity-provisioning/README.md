# User Identity Provisioning

## Overview
Demonstrated hands-on experience creating a new user account through the Microsoft 365 Admin Center. This process involved configuring user identity details, assigning a product license, creating authentication credentials, and validating successful account provisioning through user sign-in testing.

This project demonstrates practical experience with Microsoft 365 user provisioning, identity lifecycle management, account configuration, and administrative validation in a cloud-based identity management environment.

---

## Environment / Tech Stack
- Microsoft 365 Admin Center
- Microsoft Entra ID
- Microsoft 365 Business Standard
- User Account Administration
- Identity and Access Management (IAM)

---

## User Provisioning Administration
- Accessed **Active Users** within Microsoft 365 Admin Center
- Initiated **Add a User** workflow
- Configured first name, last name, display name, and username
- Created authentication credentials for the new user account
- Assigned a **Microsoft 365 Business Standard** product license
- Validated successful account provisioning through authentication testing

---

## Key Skills Demonstrated
- Microsoft 365 Administration
- User Provisioning
- User Account Configuration
- Identity Lifecycle Management
- Identity and Access Management (IAM)
- Product License Assignment
- Authentication Validation

---

## Key Takeaways
User provisioning is a core administrative function in identity management. Proper account creation and licensing ensure organizational users receive the necessary access and resources while supporting standardized onboarding and secure account lifecycle management.
