# User Access Restriction Administration

## Overview
Demonstrated hands-on experience managing user authentication access by blocking sign-in for an active Microsoft 365 user account through the Microsoft 365 Admin Center. This process involved modifying user access controls and validating the restriction by testing authentication against the affected account.

This project demonstrates practical experience with Microsoft 365 user access management, account restriction enforcement, and authentication control validation in an enterprise cloud environment.

---

## Environment / Tech Stack
- Microsoft 365 Admin Center
- Microsoft Entra ID
- User Account Administration
- Identity and Access Management (IAM)
- Authentication Access Control

---

## Access Control Administration
- Accessed **Active Users** within Microsoft 365 Admin Center
- Selected an existing user account
- Applied **Block Sign-In** restriction
- Saved administrative changes
- Tested user authentication to validate access restriction enforcement
- Verified account lockout status through sign-in validation

---

## Key Skills Demonstrated
- Microsoft 365 Administration
- User Access Control
- Identity and Access Management (IAM)
- Account Restriction Enforcement
- Authentication Management
- Administrative Access Validation
- Enterprise Identity Security

---

## Key Takeaways
Blocking user sign-in is a critical administrative function used for access control, account suspension, and security response. This process supports enterprise identity security by restricting unauthorized access, enforcing organizational policy, and validating authentication controls during user lifecycle management.
