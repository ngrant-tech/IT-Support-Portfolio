# Multi-Factor Authentication Reset

## Overview
Demonstrated hands-on experience resetting Multi-Factor Authentication (MFA) registration settings through the Microsoft 365 Admin Center. This process involved requiring an existing user to re-register authentication methods and validating the reset by logging into the user account and completing the Microsoft Authenticator setup process again.

This project demonstrates practical experience with Microsoft 365 authentication security, MFA reset administration, user identity verification, and account security validation in a cloud-based identity management environment.

---

## Environment / Tech Stack
- Microsoft 365 Admin Center
- Microsoft Entra ID
- Microsoft Authenticator
- Multi-Factor Authentication (MFA)
- Identity and Access Management (IAM)

---

## Multi-Factor Authentication Reset Administration
- Accessed **Multi-Factor Authentication** settings within Microsoft 365 Admin Center
- Selected an existing user account
- Opened **User MFA Settings**
- Required the user to provide authentication contact methods again
- Saved administrative changes
- Validated successful MFA reset through user authentication testing

---

## Key Skills Demonstrated
- Microsoft 365 Administration
- Multi-Factor Authentication (MFA) Administration
- User Authentication Security
- Identity and Access Management (IAM)
- Microsoft Authenticator Configuration
- Authentication Reset Management
- Security Validation

---

## Key Takeaways
Resetting MFA registration is an important security function used during device changes, compromised authentication methods, or account recovery scenarios. Proper MFA reset administration ensures users can securely re-establish authentication methods while maintaining account protection standards.
